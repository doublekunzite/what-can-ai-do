How I Made This Website

Date: 2024-12-01 | Model: Kimi K2

Background

This shows the original conversation that led to creating this entire website. It demonstrates how AI can help plan, structure, and execute a complete project from scratch.

I had next to no coding experience and had the website live in 15 minutes by following Kimi's guidance.

Initial Prompts

View Planning Conversation

User:

help me create an educational website about AI. the name i have in mind for now is "what can I do with AI?". i'm envisioning something quite minimalist and bare bones in terms of design. these are some of the features i want it to have:

a page that lists possible uses of AI and examples of each use case. the idea is to help people think about possible uses that might not have occurred to them.

a page about the risks, dangers, and other challenges of using AI

a page comparing different AI LLMs, how they perform on various metrics

a page titled something like "Isn't AI bad?" which will address common talking points against AI

Kimi K2:

Design Philosophy (Minimalist & Bare Bones)
  • Single-column layout with generous white space
  • Monochromatic color scheme (black text, white background, one accent color like #0066cc)
  • System fonts only
  • No images unless absolutely necessary
  • No JavaScript frameworks‚Äîvanilla JS only where essential
Page-by-Page Content Strategy

Homepage: Simple hero with clear navigation and 2-sentence descriptions of each section.

Use Cases Page: Organize by domains (Creative, Analytical, Communication, Productivity, Learning, Professional). For each use case, include sample prompts people can copy and try immediately.

Risks Page: Organize by severity and mitigation. Include Privacy & Data, Accuracy & Hallucinations, Bias & Fairness, Dependency & Skill Loss, Environmental Impact, Economic Disruption.

LLM Comparison Page: Comparison table with 4-5 major models. Add a "Which model should I use?" decision tree.

"Isn't AI Bad?" Page: Frame as a dialogue. List common objections with nuanced, non-defensive responses.

Technical Implementation

File structure with 5 HTML pages, one CSS file, and optional JS for copy-to-clipboard functionality.

Key Takeaways

  • AI can plan entire project structures from a simple description
  • Specificity in prompts ("minimalist and bare bones") guides better design decisions
  • Requesting examples and decision trees makes content more actionable
  • Security is built-in by keeping things simple (static HTML, no external dependencies)

Results

Following Kimi's guidance, I created a complete 5-page website with clear navigation, minimalist design, and educational content. The entire process took 15 minutes from conversation to live site using GitHub Pages.
