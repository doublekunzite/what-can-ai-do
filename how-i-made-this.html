The Philosophy

I wanted an educational site that was:

  • Truly minimalist: No bloat, no tracking, no unnecessary features
  • Secure by default: No server-side code, no database, no attack surface
  • Free to host: Zero ongoing costs
  • Easy to maintain: No updates, plugins, or security patches needed

WordPress would have added 20+ database tables and constant security updates for a simple 5-page site. That made no sense.

Tech Stack

  • HTML5: Semantic, accessible markup
  • CSS3: ~100 lines of custom styles, no frameworks
  • GitHub Pages: Free hosting with automatic HTTPS
  • GitHub Desktop: No command line needed

Total page size: Under 50KB. Loads in milliseconds.

Step-by-Step Process

1. Design First

I sketched the structure on paper first—5 pages, single-column layout, dropdown sections for organization.

2. Write the Content

Created all text content in a markdown file first, then converted to HTML. This let me focus on substance before code.

3. Build HTML/CSS

Hand-wrote semantic HTML (no div soup) and minimalist CSS. No external dependencies—everything is self-contained.

4. Security Audit

Before deploying, I manually checked every line of code for:

  • No external script calls
  • No hidden tracking
  • No user data collection
  • All links point to local files only

5. Deploy on GitHub Pages

Dragged the folder into GitHub Desktop, committed, and enabled Pages in repository settings. The site was live in 2 minutes.

Key Decisions

Why No JavaScript Framework?

Didn't need React for a static site. Native HTML `

` elements provide dropdown functionality without any JS.

Why No External Fonts or CDNs?

Every external call is a security risk and a potential failure point. System fonts are fast and free.

Why Manual Table Conversion?

Copying tables from AI chats requires converting to HTML `

` syntax. It's extra work but ensures perfect formatting and accessibility.

The AI Collaboration

I used AI (Kimi) to:

  • Generate initial HTML/CSS structure
  • Debug layout issues
  • Convert markdown to HTML tables
  • Brainstorm content organization

But I manually audited every line of code before deploying. AI is an accelerator, not a replacement for understanding.
